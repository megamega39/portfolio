## サービス概要
フードデリバリーの配達員向けに、案件の単価を地図を介して共有できるWebアプリです。
ユーザーが受けたオファーの単価と位置情報をリアルタイムで地図に登録・共有できます。
6000円以上の超高単価案件はクジラ、3000〜6000円の案件はマグロのアイコンで表示します。

## このサービスへの思い・作りたい理由
フードデリバリー配達員として働く中で、「今日の単価がどのくらいか」「高単価案件がどのエリアで発生しているか」を
共有できるツールがあれば便利だと感じました。
特にマグロやクジラと呼ばれる超高単価案件は発生条件が限られており、
その情報を可視化・共有できるアプリがあれば配達員同士の情報共有がもっと面白くなると思いました。

## ユーザー層について
フードデリバリー配達員
日々の単価や高単価案件の発生エリアを把握・共有したい人

## サービスの利用イメージ
アプリを開くと、地図上に他の配達員が登録した案件の単価や位置情報がピンとして表示されます。
ピンをクリックすると詳細情報（報酬額・配達距離・時間帯・天候）を確認できます。
投稿内容はリアルタイムで他のユーザーにも反映され、高単価エリアの動きを視覚的に把握できます。

## ユーザーの獲得について
フードデリバリー配達員が多く集まるSNSコミュニティ（特にX / 旧Twitter）で紹介し、口コミでの拡散を狙います。
また、既存の配達員グループへの参加やレビュー投稿を通じて利用を促進します。

## サービスの差別化ポイント・推しポイント
現在はX（Twitter）上で個々に単価を投稿・共有していますが、
本サービスでは地図上で複数ユーザーの単価情報を一括して可視化できます。
これにより、時間やエリアごとの傾向を直感的に把握できる点が差別化ポイントです。

## 機能候補

### MVPリリース時に作りたいもの:
地図上にピンと単価（金額）を登録して表示
6000円以上の案件はクジラ、3000〜5999円はマグロアイコンで表示
ピンをクリックすると詳細情報を表示（時間帯・金額・配達距離・天候）
地図ズームアウト時にはピンを非表示にして視認性を確保
10分ごとに定期的にデータを更新（ポーリング方式）

### 本リリースまでに作りたいもの:
ユーザー登録機能（Devise + JWT）
ログインユーザーによる投稿管理（自分の投稿の一覧・削除）

## 使用する技術スタック

### バックエンド:
Ruby on Rails 7.2.1（APIモード）
地図上のピン情報・ユーザー情報をJSON形式で提供するAPIサーバーとして構築します。

### データベース:
PostgreSQL 15
位置情報（緯度・経度・金額・投稿時間など）を格納します。
将来的にPostGIS拡張を導入し、エリア分析などに活用できるようにします。

### フロントエンド:
JavaScript（ES6） + Leaflet.js
OpenStreetMapを利用した地図表示を行い、ピン設置・金額ラベル・詳細ポップアップを描画します。
ピン情報はポーリング方式で定期的にデータを更新します。
カスタムアイコンを使用し、マグロ🐟やクジラ🐋などのデザインを実現予定です。

Tailwind CSS
地図操作ボタンや投稿フォームをシンプルで統一感のあるUIで構築します。

### ユーザー認証:
Devise（ユーザー管理）＋ JWT（API認証）
　Deviseを用いてユーザー登録・ログイン機能を実装し、APIモードではdevise-jwtを導入してトークン認証を行います。
　これにより、ログイン中のユーザーのみがピン登録や削除を行えるようになります。
　将来的にはメール認証・匿名ログイン・SNS認証への拡張も想定しています。


### 地図API:
OpenStreetMap（OSM） + Leaflet.js
完全無料の地図データを利用し、Leafletを通してブラウザ上で地図を表示します。
ピン設置・クリック時ポップアップ・金額ラベル表示などをJSで制御します。

### デプロイ:
Fly.io
Railsアプリ・PostgreSQLをFly.io上でデプロイします。
